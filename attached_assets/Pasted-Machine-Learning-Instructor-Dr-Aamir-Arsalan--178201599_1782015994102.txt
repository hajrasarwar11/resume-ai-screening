Project Title: Developing and Optimizing Machine Learning Models for Classification, Regression, or Clustering Tasks Using Real-World Datasets  Project Statement: This project challenges students to develop and optimize machine learning models that address real-world problems by applying both supervised and unsupervised learning techniques. Each student/group will identify a problem domain—such as healthcare, finance, education, e-commerce, or agriculture—and select an appropriate dataset. They will perform end-to-end implementation starting from data preprocessing to model building, hyperparameter tuning, and performance evaluation. The ultimate goal is to produce not only a technically sound model but also actionable insights from the results. Students will also explore trade-offs in model complexity vs. interpretability, bias vs. variance, and performance vs. computational efficiency.   Objectives: 1. Enable students to apply supervised and unsupervised ML models to real-world problems. 2. Equip students with skills to clean, explore, and transform data before modeling. 3. Teach techniques for rigorous model evaluation using domain-specific metrics. 4. Promote analytical thinking through performance optimization and tuning. 5. Encourage practical use of tools like scikit-learn, pandas, matplotlib, and seaborn. CLO Mapping: The CLO covered by the project are:  • CLO2: Apply supervised and unsupervised machine learning techniques to solve classification, regression or clustering problems. • CLO3: Analyze machine learning models and improve its performance by hyperparameter tuning.  PLO Mapping: The PLOs to which the CLO of this project has been mapped are:   • Problem Analysis (PLO2): An ability to identify, formulate, research literature, and analyze complex engineering problems reaching substantiated conclusions using first principles of mathematics, natural sciences and engineering sciences.  • Investigation (PLO4): An ability to investigate complex engineering problems in a methodical way including literature survey, design and conduct of experiments, analysis and interpretation of experimental data, and synthesis of information to derive valid conclusions.  • Modern Tool Usage (PLO5): An ability to create, select and apply appropriate techniques, resources, and modern engineering and IT tools, including prediction and modeling, to complex engineering activities, with an understanding of the limitations.  Project Phases: Phase 1: Problem Selection, Dataset Collection and Preprocessing (Mapped to WP2 – Range of Conflicting Requirements) • Identify a domain problem and define a clear objective (classification, regression, or clustering). • Choose a dataset (Kaggle, UCI ML Repository, Data.gov, WHO, etc.) or gather data manually. • Preprocess data: o Handle missing values (imputation, deletion strategies). o Remove or treat outliers. o Normalize, scale, or encode features. o Feature engineering and dimensionality reduction if needed. • Visualize feature distributions and correlations.  Phase 2: Supervised and Unsupervised Model Implementation (Mapped to WP1 – Depth of Knowledge, WP3 – Depth of Analysis) • Apply supervised learning for classification or regression tasks: o Algorithms can include Logistic Regression, Decision Trees, Random Forest, Gradient Boosting (XGBoost/LightGBM), SVM, k-NN. • Apply unsupervised learning for clustering: o Algorithms: K-Means, DBSCAN, Hierarchical Clustering, Gaussian Mixture Models. • Implement multiple models and compare performance. • Justify the selection of algorithms based on data type, size, and objective.  Phase 3: Evaluation and Visualization (Mapped to WP1 – Depth of Knowledge, WP3 – Depth of Analysis) • Apply evaluation metrics: o Classification: Accuracy, Precision, Recall, F1-score, ROC-AUC. o Regression: RMSE, MAE, R² Score. o Clustering: Silhouette Score, Davies–Bouldin Index. • Visualize results: o Confusion matrices, ROC curves, cluster plots, feature importances. Phase 4: Hyperparameter Tuning and Optimization (Relevant WPs: WP1 (Depth of Knowledge Required), WP2 (Range of Conflicting Requirements), WP3 (Depth of Analysis Required))  • Perform tuning using: o Grid Search o Randomized Search o Bayesian Optimization (optional) • Improve model generalization: o Cross-validation (k-Fold CV) o Regularization (L1, L2) o Ensemble methods  Mapping to Complex Engineering Problems (CEPs) 1. WP1: Depth of Knowledge Required o Requires applying in-depth ANN concepts and forefront engineering knowledge. 2. WP2: Range of Conflicting Requirements o Requires balancing trade-offs between model complexity, computational efficiency, and datasetspecific challenges.   3. WP3: Depth of Analysis Required o Involves abstract thinking to analyze datasets, model architectures, and problem-specific solutions.

# AIRECRUIT Information Architecture & Feature Placement Rules

You are building an enterprise-grade AI recruitment intelligence platform called **AIRECRUIT**. Think like a senior UX architect and SaaS product designer, not like a student project generator.

## PRIMARY OBJECTIVE

Create a highly organized, premium, and intelligent application where:

* Every feature has one home only.
* No duplicate functionality exists anywhere.
* No page overlaps with another.
* Users should immediately understand where everything belongs.
* The platform should feel structured, elegant, and world-class.

---

# STRICT SIDEBAR STRUCTURE

## 🏠 Home

Purpose: Platform overview only.

Contains:

* Hero section
* Project introduction
* Key highlights
* Quick statistics
* Architecture overview
* Feature cards
* System status

Never place:

* Resume upload
* Analytics
* Charts
* Predictions
* Model metrics

---

# GOLDEN RULES

❌ No duplicate cards.

❌ No duplicate charts.

❌ No repeated tables.

❌ No same functionality in two sections.

❌ No feature clutter.

❌ No random placement.

❌ No oversized pages with mixed responsibilities.

---

# DESIGN PHILOSOPHY

Think like:

* OpenAI
* Stripe
* Palantir
* Bloomberg Terminal
* Datadog

The UI should feel luxurious, premium, structured, and extremely intuitive.

Every section must answer only one question:

Home → What is AIRECRUIT?

Workflow → How does it work?

Resume Intelligence → What does this resume contain?

Recommendation Engine → Who is the best candidate?

ML Models → How do the algorithms work?

Candidate Analytics → What insights exist in the data?

Performance Metrics → Which model performs best?

About → Who built the system?

Before adding any component, ask:

"Does this belong here, and nowhere else?"

If not, place it in its correct section.

Quality and structure are more important than quantity.

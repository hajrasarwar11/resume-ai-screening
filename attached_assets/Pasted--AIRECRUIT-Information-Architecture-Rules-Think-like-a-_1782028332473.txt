# AIRECRUIT — Information Architecture Rules

Think like a senior UX architect and SaaS product designer. The application must follow strict separation of responsibilities. Every feature must have exactly one home and must never appear again elsewhere.

## Global Principles

* No duplicate functionality.
* No duplicate charts, cards, or explanations.
* Every page answers a different question.
* Avoid feature overlap.
* Keep the navigation elegant and enterprise-grade.
* Prioritize clarity over quantity.
* Each section must feel independent.

## GOLDEN RULE

Before adding any component ask:

"Does this feature already exist somewhere else?"

If YES → do not add it.

Every feature must exist only once.

The platform should feel like OpenAI, Palantir, Stripe, or Bloomberg Terminal—not a crowded student dashboard.

Section 01

Executive Summary

This report documents the end-to-end development of ResumeAI — an AI-powered intelligent resume screening and candidate-role recommendation system built as a university Machine Learning project. The system addresses a critical bottleneck in modern recruitment: the manual and time-consuming process of reviewing hundreds of resumes to identify qualified candidates for specific job roles.

The solution combines Natural Language Processing (NLP) techniques, TF-IDF vectorization, supervised classification models, and an unsupervised clustering layer to fully automate the pipeline from raw resume ingestion to ranked candidate output. A production-quality Streamlit web application exposes the system to end users.

Section 02

Project Overview

Problem Statement

Traditional recruitment pipelines are manual, slow, and prone to unconscious bias. Hiring managers spend an average of 6–7 seconds reviewing a single resume, resulting in qualified candidates being overlooked and unqualified ones advancing. Organisations managing high-volume applications face compounding inefficiencies that delay time-to-hire and increase operational costs.

Objectives

  • Automate resume parsing and textual preprocessing at scale.
  • Extract structured skills and qualifications from unstructured resume text.
  • Classify candidates into predefined job-role categories using ML classifiers.
  • Compute semantic similarity between candidate profiles and job descriptions.
  • Generate ranked shortlists using a composite scoring formula.
  • Deploy an interactive, production-quality web application for end users.

Expected Outcomes

Efficiency Gain
Automated Screening
Reduce manual resume review time by automating first-pass screening through NLP classification.
Accuracy
Precision Role Matching
Achieve ≥95% classification accuracy through hyperparameter-tuned SVM and ensemble models.
Recommendations
Ranked Candidate Lists
Produce ordered shortlists for any job description using a configurable composite scoring engine.
Deployment
Interactive Dashboard
Deliver results through a clean, real-time Streamlit dashboard accessible without technical expertise.

Section 05

System Workflow

The system operates as a structured eight-stage pipeline that takes unstructured resume text as input and produces an ordered, scored list of recommended candidates as output.

1
Job Requirement Input
The hiring manager specifies a job role by selecting from the jobs corpus or pasting a free-text job description. The description is stored as a query vector for downstream matching.
2
Resume Upload & Ingestion
Candidate resumes are ingested from the pre-processed corpus (CSV). Future iterations support real-time PDF upload via pdfplumber and PyMuPDF.
3
Text Preprocessing
Raw resume text is lowercased, tokenised, stop-word filtered, and lemmatised using NLTK. Noise characters, URLs, and special symbols are removed to produce a clean text representation.
4
Feature Engineering & Skill Extraction
Structured features — experience years, education level, skill count, skill list — are extracted. TF-IDF vectorisation (3 000 features) converts text into a numeric matrix. A separate TF-IDF with 5 000 features is trained for semantic matching.
5
Machine Learning Classification
The best-performing model (SVM) predicts the job-role category for each resume. A ColumnTransformer Pipeline ensures TF-IDF is fitted only on training data, preventing data leakage across CV folds.
6
K-Means Candidate Clustering
Candidates are grouped into k=10 clusters using K-Means on the TF-IDF matrix. Cluster quality is validated using Silhouette Score, Elbow Method, and Davies-Bouldin Index.
7
Role Recommendation & Matching
Cosine similarity is computed between the job description vector and every resume vector. Each candidate receives a percentage Match Score reflecting semantic alignment with the role.
8
Composite Ranking & Output
Candidates are ranked using a weighted composite formula. The top-N candidates are returned with full profile details, scores, and a visual bar chart.
Section 06

NLP & Feature Engineering

Step 1
Text Preprocessing
Lowercase normalisation → tokenisation → stop-word removal → lemmatisation (NLTK WordNetLemmatizer) → regex cleaning of URLs, punctuation, and numeric noise.
Step 2
Skill Extraction
Keyword matching against a curated skills vocabulary (skills_list.csv) identifies technical and soft skills present in each resume. Skill count is derived as a numeric feature.
Step 3
TF-IDF Vectorisation (Classification)
max_features=3,000 — fitted strictly on training data inside a ColumnTransformer Pipeline to prevent leakage. Produces a sparse matrix used as input to all classifiers.
Step 4
TF-IDF Vectorisation (Matching)
max_features=5,000 — a second TF-IDF fitted on the full corpus for semantic matching. The shared vocabulary enables apples-to-apples cosine similarity between resumes and job descriptions.

Data Leakage Prevention

Critical Design Constraint: All TF-IDF vectorisers are wrapped inside sklearn.pipeline.Pipeline with ColumnTransformer. This ensures that during cross-validation each fold refits the vectoriser only on its training portion — preventing test-set vocabulary from leaking into the model. Train/test split always precedes any feature fitting step.

Structured Feature Engineering

  • Experience Years — numeric, normalised 0–100 for ranking
  • Education Level — ordinal encoded: 0=High School, 1=Bachelor's, 2=Master's, 3=PhD
  • Skill Count — count of matched skills from vocabulary, normalised 0–100 for ranking
  • Category Encoded — integer label for 42 job-role classes
  • Cluster ID — K-Means cluster assignment (k=10)
Section 07

Machine Learning Models

Four supervised classifiers were trained, evaluated, and compared. In addition, K-Means unsupervised clustering was used to discover latent candidate segments within the resume corpus.

Classifier 1
Support Vector Machine (SVM) BEST
Linear SVM with TF-IDF pipeline. Excels at high-dimensional text classification. Best accuracy at 98.75%. Saved as svm_pipeline.pkl.
Classifier 2
Random Forest
Ensemble of decision trees. Tuned using GridSearchCV (cv=5) over n_estimators, max_depth, and min_samples_split. Saved as random_forest_tuned_gridsearch.pkl.
Classifier 3
Logistic Regression
Evaluated with both L1 (Lasso, sparse) and L2 (Ridge, shrinkage) regularisation at C ∈ {0.01, 0.1, 1.0, 10.0} using the saga solver.
Classifier 4
XGBoost
Gradient-boosted trees tuned with RandomizedSearchCV (n_iter=20, cv=3) over continuous distributions of learning rate, max_depth, and n_estimators. Saved as xgboost_tuned_randomsearch.pkl.
Unsupervised
K-Means Clustering
Applied to the TF-IDF resume matrix. Optimal k=10 selected via Elbow Method (inertia), Silhouette Score (cohesion/separation), and Davies-Bouldin Index (cluster overlap). Saved as kmeans.pkl.
Cross-Validation
Stratified K-Fold
Stratified splitting preserves class proportions across folds. Hyperparameter search uses cv=5 (GridSearch) and cv=3 (RandomizedSearch) to balance variance estimation with compute cost.

Section 09

Recommendation Engine

The recommendation engine computes semantic similarity between any job description and all candidate resumes using TF-IDF cosine similarity, then filters and sorts results by match score.

Input
Job Description Query
Hiring manager selects a job from the corpus or pastes a custom description. The text is vectorised using the shared 5 000-feature TF-IDF vocabulary.
Matching
Cosine Similarity
Cosine similarity is computed between the JD vector and every resume vector. Returns a percentage Match Score reflecting text-level alignment between candidate and role.
Filter
Category-Based Filtering
Optionally filter candidates by predicted role category before similarity computation. Reduces noise and improves relevance of shortlist.
Output
Ranked Shortlist
Top-N candidates returned with Resume ID, Category, Match Score, experience, skills, and education level displayed in the Streamlit dashboard.
Section 10

Candidate Ranking System

Beyond simple text similarity, the ranking system produces a composite score that balances three signals: semantic match, professional experience, and technical breadth.

Final Score = (Wmatch / 100) × Match Score
             + (Wexp / 100) × Normalised Experience
             + (Wskill / 100) × Normalised Skill Count

Default: Wmatch=60 Wexp=20 Wskill=20
60%
Cosine Match Score
TF-IDF semantic similarity between resume and job description text.
20%
Normalised Experience
Years of professional experience normalised to 0–100 scale against the corpus maximum.
20%
Normalised Skill Count
Number of matched skills from the curated vocabulary, normalised to 0–100.
Configurable Weights: The Streamlit ranking page allows hiring managers to adjust Wmatch (40–80%) and Wexp (5–40%) via interactive sliders. Skill weight auto-calculates as the remainder. This enables role-specific calibration (e.g. weight experience more heavily for senior roles).

Section 14

Conclusion

The AI-Powered Intelligent Resume Screening and Candidate Role Recommendation System successfully demonstrates the practical application of Machine Learning and Natural Language Processing to a real-world recruitment problem. The system achieves a peak classification accuracy of 98.75% with the Support Vector Machine classifier — well above the project's initial 95% target threshold.

The composite ranking engine, combining cosine similarity, experience normalisation, and skill matching, provides a nuanced, configurable candidate scoring mechanism that goes beyond simple keyword matching. The full-stack Streamlit deployment exposes the pipeline through an intuitive, visually polished interface accessible without technical expertise.

The modular notebook architecture — from EDA through preprocessing, model training, and evaluation — ensures full reproducibility and a clear audit trail for each analytical decision. All models are serialised as Scikit-Learn Pipelines to prevent data leakage and enable safe production deployment.

Future Work

  • BERT or sentence-transformer embeddings to replace TF-IDF for deeper semantic understanding
  • Bias detection and fairness analysis across demographic groups

  • Continuous learning pipeline with human-in-the-loop feedback
  • Multi-language resume support via multilingual NLP models

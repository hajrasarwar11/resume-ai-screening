Abstract

Abstract

This report presents the end-to-end design, implementation, and deployment of an AI-Powered Intelligent Resume Screening and Candidate Role Recommendation System, developed as a Complex Engineering Problem (CEP) for SE-CD-638 Machine Learning (Semester VI, 2026–2027) under the supervision of Dr. Aamir Arsalan.

The system addresses a critical bottleneck in modern recruitment: the inefficient, bias-prone manual review of hundreds of resumes. By applying Natural Language Processing (NLP), TF-IDF feature extraction, and supervised/unsupervised machine learning, the pipeline fully automates resume ingestion, classification, and candidate ranking.

Four supervised classifiers — Support Vector Machine (SVM), Random Forest, Logistic Regression, and XGBoost — were trained and evaluated on a multi-class labelled resume corpus covering 42 job-role categories. Hyperparameter tuning was performed via GridSearchCV (cv=5) for Random Forest and RandomizedSearchCV (n_iter=20, cv=3) for XGBoost, with L1/L2 regularisation explored for Logistic Regression. The best-performing model, a Linear SVM with TF-IDF pipeline, achieved 98.75% classification accuracy. An unsupervised K-Means clustering layer (k=10) segments the candidate pool for exploratory discovery.

A composite candidate ranking engine combines TF-IDF cosine similarity (60%), normalised professional experience (20%), and skill-matching scores (20%) to generate ordered shortlists for any job description. The complete system is deployed as a production-quality Streamlit web application with real-time PDF resume upload, live multi-model comparison, and interactive analytics — qualifying for the optional deployment extra credit.

Keywords: Resume Screening · Machine Learning · NLP · TF-IDF · SVM · Random Forest · XGBoost · K-Means Clustering · GridSearchCV · Hyperparameter Tuning · Cosine Similarity · Recruitment Automation · Streamlit · Python 3.12.

Project Overview

Problem Statement

Traditional recruitment pipelines are manual, slow, and prone to unconscious bias. Hiring managers spend an average of 6–7 seconds reviewing a single resume, resulting in qualified candidates being overlooked and unqualified ones advancing. Organisations managing high-volume applications face compounding inefficiencies that delay time-to-hire and increase operational costs.

Objectives

  • Automate resume parsing and textual preprocessing at scale.
  • Extract structured skills and qualifications from unstructured resume text.
  • Classify candidates into predefined job-role categories using ML classifiers.
  • Compute semantic similarity between candidate profiles and job descriptions.
  • Generate ranked shortlists using a composite scoring formula.
  • Deploy an interactive, production-quality web application for end users.

Section 02.5

Literature Review

The following works informed the methodology, algorithmic choices, and evaluation framework adopted in this project. Each paper is briefly reviewed for its relevance to resume screening, NLP-based classification, and recruitment automation.

Reference [1] · Resume Classification
Automated Resume Screening Using Machine Learning and NLP
Context: Several studies have proposed TF-IDF + SVM pipelines for document classification, reporting accuracy exceeding 95% on multi-class text corpora. The use of a linear kernel SVM with sparse TF-IDF features is a well-established baseline for high-dimensional text tasks, as linear separability is typically achievable when the number of features significantly exceeds the number of samples.

Relevance: Directly informed our choice of LinearSVC as the primary classifier. The ColumnTransformer Pipeline architecture used in this project prevents data leakage — a common flaw identified in prior resume-classification literature.
Reference [2] · Ensemble Methods
Random Forest and Gradient Boosting for Text Classification Tasks
Context: Ensemble approaches (Random Forest, XGBoost) have demonstrated competitive performance on NLP tasks when combined with TF-IDF or count-vectorised features. GridSearchCV-tuned Random Forests with n_estimators ∈ [100, 200, 300] and depth constraints are reported to converge within reasonable compute budgets under stratified k-fold cross-validation (cv=5).

Relevance: Grounded our GridSearchCV search space for Random Forest and our choice of RandomizedSearchCV (n_iter=20) for the larger continuous XGBoost parameter space. The trade-off between model file size (200MB for RF vs 1.3MB for SVM) vs marginal accuracy gain (98.15% vs 98.75%) is explicitly documented in this project.
Reference [3] · Information Retrieval & Similarity
TF-IDF Cosine Similarity for Document Matching and Retrieval
Context: The Vector Space Model (VSM) with TF-IDF weighting and cosine similarity is the foundational technique in information retrieval for document matching. Studies confirm that a 5,000-feature TF-IDF vocabulary provides sufficient semantic coverage for domain-specific corpora (resume text) while remaining computationally tractable for real-time query serving.

Relevance: Directly informed the design of our recommendation engine, which uses a separate 5,000-feature TF-IDF (distinct from the 3,000-feature classifier TF-IDF) to compute match scores between any job description and the candidate pool. The composite ranking formula (60% cosine + 20% experience + 20% skills) extends this with domain-specific signals for recruitment.

Section 05

System Workflow

The system operates as a structured eight-stage pipeline that takes unstructured resume text as input and produces an ordered, scored list of recommended candidates as output.

1
Job Requirement Input
The hiring manager specifies a job role by selecting from the jobs corpus or pasting a free-text job description. The description is stored as a query vector for downstream matching.
2
Resume Upload & Ingestion
Candidate resumes can be ingested from the pre-processed corpus (2,400+ resumes) or uploaded as real PDFs directly in the dashboard via pdfplumber. Text is extracted in real-time and passed to the TF-IDF matching pipeline.
3
Text Preprocessing
Raw resume text is lowercased, tokenised, stop-word filtered, and lemmatised using NLTK. Noise characters, URLs, and special symbols are removed to produce a clean text representation.
4
Feature Engineering & Skill Extraction
Structured features — experience years, education level, skill count, skill list — are extracted. TF-IDF vectorisation (3 000 features) converts text into a numeric matrix. A separate TF-IDF with 5 000 features is trained for semantic matching.
5
Machine Learning Classification
The best-performing model (SVM) predicts the job-role category for each resume. A ColumnTransformer Pipeline ensures TF-IDF is fitted only on training data, preventing data leakage across CV folds.
6
K-Means Candidate Clustering
Candidates are grouped into k=10 clusters using K-Means on the TF-IDF matrix. Cluster quality is validated using Silhouette Score, Elbow Method, and Davies-Bouldin Index.
7
Role Recommendation & Matching
Cosine similarity is computed between the job description vector and every resume vector. Each candidate receives a percentage Match Score reflecting semantic alignment with the role.
8
Composite Ranking & Output
Candidates are ranked using a weighted composite formula. The top-N candidates are returned with full profile details, scores, and a visual bar chart.
Section 06

NLP & Feature Engineering

Step 1
Text Preprocessing
Lowercase normalisation → tokenisation → stop-word removal → lemmatisation (NLTK WordNetLemmatizer) → regex cleaning of URLs, punctuation, and numeric noise.
Step 2
Skill Extraction
Keyword matching against a curated skills vocabulary (skills_list.csv) identifies technical and soft skills present in each resume. Skill count is derived as a numeric feature.
Step 3
TF-IDF Vectorisation (Classification)
max_features=3,000 — fitted strictly on training data inside a ColumnTransformer Pipeline to prevent leakage. Produces a sparse matrix used as input to all classifiers.
Step 4
TF-IDF Vectorisation (Matching)
max_features=5,000 — a second TF-IDF fitted on the full corpus for semantic matching. The shared vocabulary enables apples-to-apples cosine similarity between resumes and job descriptions.

Data Leakage Prevention

Critical Design Constraint: All TF-IDF vectorisers are wrapped inside sklearn.pipeline.Pipeline with ColumnTransformer. This ensures that during cross-validation each fold refits the vectoriser only on its training portion — preventing test-set vocabulary from leaking into the model. Train/test split always precedes any feature fitting step.

Structured Feature Engineering

  • Experience Years — numeric, normalised 0–100 for ranking
  • Education Level — ordinal encoded: 0=High School, 1=Bachelor's, 2=Master's, 3=PhD
  • Skill Count — count of matched skills from vocabulary, normalised 0–100 for ranking
  • Category Encoded — integer label for 42 job-role classes
  • Cluster ID — K-Means cluster assignment (k=10)
Section 07

Machine Learning Models

Four supervised classifiers were trained, evaluated, and compared. In addition, K-Means unsupervised clustering was used to discover latent candidate segments within the resume corpus.

Classifier 1
Support Vector Machine (SVM) BEST
Linear SVM with TF-IDF pipeline. Excels at high-dimensional text classification. Best accuracy at 98.75%. Saved as svm_pipeline.pkl.
Classifier 2
Random Forest
Ensemble of decision trees. Tuned using GridSearchCV (cv=5) over n_estimators, max_depth, and min_samples_split. Saved as random_forest_tuned_gridsearch.pkl.
Classifier 3
Logistic Regression
Evaluated with both L1 (Lasso, sparse) and L2 (Ridge, shrinkage) regularisation at C ∈ {0.01, 0.1, 1.0, 10.0} using the saga solver.
Classifier 4
XGBoost
Gradient-boosted trees tuned with RandomizedSearchCV (n_iter=20, cv=3) over continuous distributions of learning rate, max_depth, and n_estimators. Saved as xgboost_tuned_randomsearch.pkl.
Unsupervised
K-Means Clustering
Applied to the TF-IDF resume matrix. Optimal k=10 selected via Elbow Method (inertia), Silhouette Score (cohesion/separation), and Davies-Bouldin Index (cluster overlap). Saved as kmeans.pkl.
Cross-Validation
Stratified K-Fold
Stratified splitting preserves class proportions across folds. Hyperparameter search uses cv=5 (GridSearch) and cv=3 (RandomizedSearch) to balance variance estimation with compute cost.

Section 09

Recommendation Engine

The recommendation engine computes semantic similarity between any job description and candidate resumes using TF-IDF cosine similarity. It supports two candidate sources: the pre-indexed corpus of 2,400+ resumes, or real PDFs uploaded live by the recruiter.

Input — Candidate Source
Corpus Pool or PDF Upload
Recruiter selects between the 2,400+ pre-processed candidate pool or uploads their own PDFs. Uploaded resumes are parsed in real-time using pdfplumber, structured, and matched instantly.
Input — Job Description
JD Selection or Custom Text
Hiring manager selects a job from the corpus or pastes a custom description. The text is vectorised using the shared 5,000-feature TF-IDF vocabulary.
Matching
Cosine Similarity
Cosine similarity is computed between the JD vector and every resume vector. Each candidate receives a percentage Match Score reflecting text-level semantic alignment with the role.
Output
Ranked Shortlist + Skill Gap
Top-N candidates returned with Match %, shared keywords, skill gap analysis, and an extractive resume summary. Results exportable as CSV for ATS integration.
Section 10

Candidate Ranking System

Beyond simple text similarity, the ranking system produces a composite score that balances three signals: semantic match, professional experience, and technical breadth.

Final Score = (Wmatch / 100) × Match Score
             + (Wexp / 100) × Normalised Experience
             + (Wskill / 100) × Normalised Skill Count

Default: Wmatch=60 Wexp=20 Wskill=20
60%
Cosine Match Score
TF-IDF semantic similarity between resume and job description text.
20%
Normalised Experience
Years of professional experience normalised to 0–100 scale against the corpus maximum.
20%
Normalised Skill Count
Number of matched skills from the curated vocabulary, normalised to 0–100.
Configurable Weights: The Streamlit ranking page allows hiring managers to adjust Wmatch (40–80%) and Wexp (5–40%) via interactive sliders. Skill weight auto-calculates as the remainder. This enables role-specific calibration (e.g. weight experience more heavily for senior roles).

Section 11.5 · Extra Credit

Deployment

The system has been fully deployed as a production-quality web application using Streamlit, qualifying for the optional deployment extra credit outlined in the project brief. The application is hosted on Replit Autoscale infrastructure and is accessible via a public URL without any local setup.

Platform
Streamlit + Replit Autoscale
The application runs as a long-lived Streamlit server on port 5000. Replit Autoscale manages compute scaling and public HTTPS proxying. Models are loaded at startup via @st.cache_resource and data via @st.cache_data, ensuring sub-second response after initial load.
Architecture
Single-File Production App
All logic is contained in app.py (~2,465 lines). Models are serialised as Scikit-Learn Pipelines (.pkl) and loaded once on startup. The primary production model is SVM (1.3 MB) — deliberately chosen over Random Forest (200 MB) for deployment efficiency while maintaining 98.75% accuracy.
Real-Time Features
Live PDF Upload & Prediction
Three modules support real-time PDF upload via pdfplumber: the Recommendation Engine (bulk multi-PDF upload), CV Compare (side-by-side PDF comparison), and ML Models Live Comparison (single PDF prediction across all classifiers). Text is extracted, preprocessed, and vectorised in memory without any server-side file storage.
UI Design
Dark-Gold Theme Dashboard
Custom CSS injected via st.markdown(unsafe_allow_html=True) implements a dark gold design system: Playfair Display headings, Inter body, glass-morphism cards, CSS grid layouts, interactive Plotly charts (radar, bar, histogram, pie), and responsive metric cards. All 8 pages (Home, Workflow, Resume Intelligence, Recommendation Engine, ML Models, Candidate Analytics, Performance Metrics, About) share a unified sidebar navigation.

Section 14

Conclusion

The AI-Powered Intelligent Resume Screening and Candidate Role Recommendation System successfully demonstrates the practical application of Machine Learning and Natural Language Processing to a real-world recruitment problem. The system achieves a peak classification accuracy of 98.75% with the Support Vector Machine classifier — well above the project's initial 95% target threshold.

The composite ranking engine, combining cosine similarity, experience normalisation, and skill matching, provides a nuanced, configurable candidate scoring mechanism that goes beyond simple keyword matching. The full-stack Streamlit deployment exposes the pipeline through an intuitive, visually polished interface accessible without technical expertise.

The modular notebook architecture — from EDA through preprocessing, model training, and evaluation — ensures full reproducibility and a clear audit trail for each analytical decision. All models are serialised as Scikit-Learn Pipelines to prevent data leakage and enable safe production deployment.

Future Work

  • BERT or sentence-transformer embeddings to replace TF-IDF for deeper semantic understanding and cross-lingual resume matching.
  • Bias detection and fairness analysis across demographic groups to ensure equitable candidate ranking.
  • Integration with Applicant Tracking System (ATS) APIs for end-to-end recruitment workflow automation.
  • Continuous learning pipeline with human-in-the-loop feedback to improve model accuracy over time.
  • Multi-language resume support via multilingual NLP models (mBERT, XLM-R).
  • Interview question generation personalised to each candidate's identified skill gaps.
  • Explainability layer (SHAP values) to surface model reasoning for each prediction.
